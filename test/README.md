# Vault REST API Test Scripts

Shell scripts to interact with the Vault cluster via its HTTP REST API. These scripts are meant to be run from the **host machine** and require an SSH tunnel to reach the Vault API.

## Prerequisites

- Vault cluster running and unsealed (`make up` + `make init`)
- SSH tunnel to a Vault node active on `127.0.0.1:8200`:

```bash
make ssh-tunnel
```

- `curl` and `jq` installed on the host

## Scripts

### `create-secret.sh`

Writes a secret to Vault's KV2 secrets engine.

```bash
# Single key/value pair
./test/create-secret.sh -p myapp/config -k db_password -v s3cret

# Multiple keys via JSON payload
./test/create-secret.sh -p myapp/config -d '{"user":"admin","pass":"s3cret"}'

# Custom Vault address
./test/create-secret.sh -p myapp/config -k foo -v bar -a http://10.0.10.11:8200
```

| Option | Description | Default |
|--------|-------------|---------|
| `-p PATH` | Secret path (required) | — |
| `-k KEY` | Key name | — |
| `-v VALUE` | Value (requires `-k`) | — |
| `-d DATA` | Full JSON payload (alternative to `-k`/`-v`) | — |
| `-a VAULT_ADDR` | Vault address | `http://127.0.0.1:8200` |
| `-m MOUNT` | KV2 mount path | `secret` |
| `-c FILE` | Credentials file | `../.vault-credentials.json` |
| `-t TOKEN` | Vault token (overrides credentials file) | — |

---

### `read-secret.sh`

Reads a secret from Vault's KV2 secrets engine.

```bash
# Read all keys in a secret
./test/read-secret.sh -p myapp/config

# Extract a single key
./test/read-secret.sh -p myapp/config -k db_password
```

| Option | Description | Default |
|--------|-------------|---------|
| `-p PATH` | Secret path (required) | — |
| `-k KEY` | Extract a specific key from the secret | — |
| `-a VAULT_ADDR` | Vault address | `http://127.0.0.1:8200` |
| `-m MOUNT` | KV2 mount path | `secret` |
| `-c FILE` | Credentials file | `../.vault-credentials.json` |
| `-t TOKEN` | Vault token (overrides credentials file) | — |

## Example Workflow

```bash
# 1. Open SSH tunnel (background)
make ssh-tunnel &

# 2. Write a secret
./test/create-secret.sh -p myapp/db -k password -v sup3rs3cret

# 3. Read it back
./test/read-secret.sh -p myapp/db

# 4. Read a specific key
./test/read-secret.sh -p myapp/db -k password
```

## Authentication

Both scripts authenticate using the root token from `.vault-credentials.json` (generated by `make init`). You can override this with the `-t` flag to use a different token.
